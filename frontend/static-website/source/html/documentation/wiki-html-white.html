HTML Doctype

About Doctype

The HTML !DOCTYPE declaration is the required preamble found at the top of all HTML documents before opening the html tag and it is an instruction to the web-browser about what version of HTML the page is written in

The main purpose of the !DOCTYPE declaration is to prevent a browser from switching into the so-called quirk mode when rendering the HTML document.

Also the !DOCTYPE html declaration ensures that the browser makes the best-effort attempt at following the relevant specifications rather than using a different rendering mode that is incompatible with some specifications and it is required to insure that your web pages are displayed the way they were intended to be displayed.

Nota Bene

  • The !DOCTYPE declaration is NOT a HTML tag and is NOT case sensitive;
  • The !DOCTYPE declaration points to a Document Type Definition (DTD) and it is an instruction to the web browser about the version of the markup language the page is written in;

Tips & Tricks

  • Always use W3C's validator to check your HTML for markup or syntax error before publishing online;
  • There is an additional tag to be put for Internet Explorer <=10 users to avoid quirk mode in the head: <meta http-equiv="X-UA-Compatible" content="IE-Edge">

Notes

Quirk Mode
Refers to a technique used by some web browsers for the sake of maintaining backward compatibility with web pages designed for old web browsers instead of strictly complying with W3C and IETF standards in standards mode.

Document Object Model (DOM)

The DOM is a programming interface for HTML and XML documents. It represents the document as nodes and objects which can be accessed with programming languages to change the document structure, style and content.

When the page is loaded, the browser creates a Document Object Model of the page, this represents the document that can be manipulated as it is an object-oriented representation of the page which can be modified with a programming language such as JavaScript.

The DOM is built using multiple APIs that work together, the core DOM defines the objects that describe the document and the objects within it this way other APIs can add new features and capabilities to the DOM.

To access the HTML elements with JavaScript you can select them using by ID, tag name, class name, CSS selectors and HTML object collections.

Node-trees, node elements and values

The HTML DOM Node Tree or the Document Tree is the term used to reference the HTML tree-structure which is also called a node-tree.

Through the node-tree all nodes can be accessed, their contents can be modified or deleted, and new elements can be created.

The nodes in a tree have a hierarchical relationship to each other, these relationships are described using terms such as parent, child, and sibling.

In a node tree, the top node is called the root, every node, except the root, has only one parent node, a node can have any number of children, a leaf is a node with no children and siblings are nodes with the same parent.

A node-element is the most general base class from which all element objects in a HTML document inherits properties from its parent node.

The node-value is used to get the content of the node, mostly used for text elements becaus for most of the others it returns null.

HTML5 Semantic & Sectioning Elements

Semantics in HTML

When we talk about semantics in programming we refer to the meaning of the piece of code in that specific purpose.

HTML tags tell the browser how to display the content on the page, but as these elements do not give any indication about what type of content they contain ot the role they have, semantic HTML5 elements were implemented to address the problem by defining specific tags to indicate a clear role about the content of the element.

In HTML, semantic elements give the content they wrap around a role/meaning in the entire context of the page, and by default the majority of the user agent stylesheets will style these elements, but HTML should be coded to represent the data that will be populated and not based on its default presentation styling - presentation is the responsibility of CSS.

Benefits of using HTML semantic elements:

  • Search engines rank pages according the semantic elements and using the contents as keywords.
  • People using screen readers benefit from using semantically correct implemented elements, using these elements as signposts
  • Blocks of meaningful code is much more easier to find than searching through a much of <div> elements.
  • The semantic naming of the elements symbolizes the proper custom element naming.
  • The developer benefit from using semantic elements because these suggest the type of data that should be used as content.

There are more than 100 semantic elements available but the most used are:

  • Main root - <html>
  • Document metadata - <head>, <meta>, <link>, <style> and <title>
  • Sectioning root - <body>
  • Content sectioning - <header>, <main>, <footer>, <article>, <aside>, <nav>, <section> and <h1>-<h6>

Using the HTML5 Sectioning Elements

The main Element

The <main> element should contain the main content for your web page. All of this content should be unique to the individual page, and should not appear elsewhere on the site. Any content that is repeated on multiple pages (logos, search boxes, footer, links, etc.) should no be placed within the <main> element.

The ARIA role="main" attribute indicates the significance of this element to programs that don't yet support the <main> element (such as screen readers).

The article Element

The <article> element should contain a piece of self-contained content that could be distributed outside the context of the page. This includes things like news articles, blog posts, or user comments.

There can also be nested <article> elements within one another.

The section Element

The <section> element is used to represent a group of related content. This is similar to the purpose of an <article> element with the main difference being that the content within a <section> element does not necessarily need to make sense out of the context of the page.

There might be that the <article> element is used to represent a whole, and then multiple <section> elements to represent each of the sub-topics.

If you just need to group content together for styling purposes you should use a <div> element rather than a <section> element.

The nav Element

The <nav> element is used to mark up a collection of links to external pages or sections within the current page. As well as being used for the main website navigation, the <nav> element is also a good fit for things like a table of contents or a blog-roll.

Marking up your links within a list will often make you navigation easier to use, however this is not a requirement when using a <nav> element.

The aside Element

The <aside> element is used to represent content that is tangibly related to the content surrounding it, but could be considered separate. This includes things like sidebars, groups of <nav> elements, figures and pull quotes.

The header Element

The <header> element is used to represent the introductory content to an article of web page. This will usually contain a heading element as well as some metadata that is relevant to the content, such as a post date of a news article for example. it could also contain a table of contents (within a <nav> element) foa longer document.

A <header> element will be associated with the nearest sectioning element, usually it's direct parent in he page structure.

The footer Element

The <footer> element is used to represent information about a section such as the author, copyright information, or links ro related web pages.

As with <header>, the <footer> element is associated with the nearest sectioning element.

The address Element

The <address> element is one of the most commonly misunderstood HTML elements. This element is not for making up postal address, but rather for representing the contact information for an article or web page. This could be a link to the author's website or their email address.

This element is ofthen used within the <footer> for an <article>.

Nota Bene

The ARIA role attribute is used to indicate the significance of its element to programs that do not yet support that specific semantic element, such as screen readers.

For elements that have no ARIA role assigned the aria-label attribute should be used to define the string that labels the current element.

Headers <h1> - <h6>

About

The <h1> - <h6> are used to define HTML headings and mostly used for titles and subtitles of the context they are in.

Accesibility concerns

Screen reading softweare, as a navigation technique jump from heading to heading to quickly determine the content of the page - because of this it is important not to skip heading levels because it may create confusion.

Nota Bene

Headings may be used by user agents to construct a table pf contents for a document automatically.

Heading tags should not be used for styling purposes, use CSS for that. Heading tags should be used to indicate their relative importance.

Skipping heading levels is a very bad practice.

The <h1> element should be used only once, it is the most important heading, and tells you what the purpose of the overall page is - as you would not ahve a book with more than one title per page - also having a single top-level title is arguably better for screen-reader users and SEO.

Semantic Website Structure

What is ...

HTML uses a number of block level elements that define areas of the website such as "the header", "the navigation menu", "the main content column".

It might be that webpages look pretty different from one another, but they all share the same standard components.

Which are these block level elements?

header
Usually this is he big strip across the top of the page with a big heading and/or logo. This is where the main common information about the website usually stays - this part is wrapped in the <header>.
navigation bar
All the links to the sites main sections should be found here represented by buttons, link sor tabs - this part is wrapped by <nav>.
main content
This is the biggest area of the page and contains the unique content of a given webpage - this part is wrapped by <main>, and its content divided by <article>, <section> and <div> elements.
sidebar
In this part are placed some peripheral info, links, ads, etc. Usually this is contextual to what is contained in the main content - this part is wrapped by <aside>.
footer
This is the strip across the bottom of the page that generally contains the fine print, the copyright notices or contact info. The footer is also used for SEO purposes by providing links for quick access to popular contentw- this part is wrapped by <footer>.

Meta-tags

About

Matadata is information about the data and more exactly the <meta> tag provides information about the HTML document to the browser. This metadata will not be displayed on the page but will be used by the browser, search engines and screen readers.

The <meta> elements are used to specify the page description, keywords, author of the document, how to display the content of the webpage, take control over the viewport and other metadata

Mata-related elements

There are some meta-related elements that are defined but cannot represent the <meta> element: <base>, <link>, <script>, <style> and <title>.

Understand and Code according to coding conventions

Coding conventions:

  • Always declare <!DOCTYPE html>
  • All templates must be UTF-8 encoded
  • Tags and attributes must always be declare using lowercase character.
  • Do not write multiple elements on the same line, every element should have its own line.
  • Add an additional level of indentation fo each nested element
  • Use boolean attributes without values
  • Use double quotes to quote attribute values
  • Type attributes will be omitted for link stylesheets, style and script elements
  • Always include the closing tag
  • Because of Sightly problems always include a trailing slash in self-closing elements
  • Always use alternative text for images, the only exception is when the image is decorative
  • List the attribute order as it follows: class, data-*, ID, everything else
  • Use HTML5 elements in a backward compatible way as it follows - HTML5 structure elements should only contain the role attribute, structure elements should not be used for styling, structure elements should not be referenced in JavaScript, <div> should follow as fallback.
  • Always add skiplinks to the document start
  • Do not comment your HTML code, it should be ways to understand and the documentation should be found ib the SCSS files
  • The <head> should begin with the encoding tag, the <title> should be next for SEO reasons and all the other meta-tags to follow
  • Use IDs only for the main areas of the page, to jump to site section (accessibility) or as context in JavaScript - DO NOT USE IDs for styling because of specificity reasons
